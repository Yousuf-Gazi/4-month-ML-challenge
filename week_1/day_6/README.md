## Goals

## Work Done

- Demonstrated pass-by-assignment behavior using function arguments
- Observed how local reassignment does not affect outer variables
- Implemented a function without a return value and inspected its result
- Used default function parameters for flexible configuration
- Wrote a pure function that returns a boolean result

## Key Learnings

- Function parameters are local to the function scope
- Reassigning a parameter does not mutate the original variable
- Functions without `return` implicitly return `None`
- Default parameters provide clean and flexible APIs
- Pure functions (returning values without side effects) are easier to test and reuse
